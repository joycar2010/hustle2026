# 刷新管理模块说明

## 功能概述

在系统管理页面新增"刷新管理"模块，提供统一的刷新频率管理和性能优化功能。

## 访问路径

http://13.115.21.77:3000/system → 点击"刷新管理"标签

## 核心功能

### 1. 全局设置

#### 页面可见性检测
- **功能**：自动检测页面是否可见
- **效果**：页面不可见时自动降低刷新频率
- **优势**：节省CPU、内存和网络资源
- **实现**：使用 `document.visibilityState` API

#### WebSocket 连接
- **功能**：使用WebSocket替代HTTP轮询
- **效果**：实时推送数据，减少网络开销
- **优势**：降低延迟，提升实时性
- **状态**：显示当前连接状态

#### 批量请求合并
- **功能**：将多个请求合并为一个
- **效果**：减少HTTP请求数量
- **优势**：降低服务器负载，提升性能

#### 不可见时刷新倍率
- **功能**：设置页面不可见时的刷新间隔延长倍数
- **范围**：2x - 10x
- **默认**：5x
- **示例**：原本1秒刷新，不可见时变为5秒刷新

### 2. 模块刷新频率管理

支持独立配置11个模块的刷新频率：

| 模块 | 默认频率 | 可选范围 | 说明 |
|------|---------|---------|------|
| Dashboard 价格数据 | 1秒 | 1秒-60秒/禁用 | 实时市场价格 |
| 点差数据表 | 1秒 | 1秒-60秒/禁用 | 实时点差监控 |
| 点差图表 | 1秒 | 1秒-60秒/禁用 | 盈利数据可视化 |
| 策略面板 | 1秒 | 1秒-60秒/禁用 | 策略状态更新 |
| 风险管理 | 5秒 | 1秒-60秒/禁用 | 风险指标监控 |
| 未平仓订单 | 5秒 | 1秒-60秒/禁用 | 订单列表更新 |
| 手动交易 | 5秒 | 1秒-60秒/禁用 | 最近订单刷新 |
| 日志管理 | 5秒 | 1秒-60秒/禁用 | 交易统计日志 |
| 点差历史图表 | 5秒 | 1秒-60秒/禁用 | 历史数据展示 |
| 资产仪表盘 | 10秒 | 1秒-60秒/禁用 | 资产数据刷新 |
| 账户状态面板 | 30秒 | 1秒-60秒/禁用 | 账户信息更新 |

#### 频率标签
- **极高频**：< 1秒（红色）
- **高频**：1秒（红色）
- **中频**：2-5秒（黄色）
- **低频**：> 5秒（绿色）
- **禁用**：0秒（灰色）

### 3. 实时统计

#### 活跃模块
显示当前启用的模块数量

#### 每分钟请求数
根据各模块刷新频率自动计算
- 公式：Σ(60000 / 模块刷新间隔)
- 实时更新

#### 预估流量/小时
根据请求数估算网络流量
- 假设每次请求10KB
- 公式：请求数 × 10KB × 60分钟

#### WebSocket状态
- **已连接**：绿色
- **未连接**：红色

### 4. 性能建议

系统自动分析当前配置，提供优化建议：

#### 高频模块过多
- **触发条件**：超过3个模块使用≤1秒刷新
- **建议**：启用WebSocket减少网络开销

#### 未启用可见性检测
- **触发条件**：页面可见性检测关闭
- **建议**：启用以节省资源

#### 请求数过高
- **触发条件**：每分钟请求数 > 300
- **建议**：适当降低部分模块刷新频率

#### 未启用批量请求
- **触发条件**：批量请求关闭且高频模块 > 2
- **建议**：启用批量请求合并

#### 配置良好
- **触发条件**：无任何问题
- **显示**：当前配置良好，系统运行在最佳状态

## 技术实现

### 前端

#### 文件位置
`frontend/src/views/System.vue`

#### 核心功能
1. **配置管理**
   - 加载/保存刷新设置
   - 实时计算统计数据
   - 生成性能建议

2. **事件广播**
   ```javascript
   window.dispatchEvent(new CustomEvent('refresh-settings-changed', {
     detail: { settings, modules }
   }))
   ```

3. **响应式计算**
   - activeModulesCount
   - requestsPerMinute
   - estimatedTrafficPerHour
   - performanceSuggestions

### 后端

#### 文件位置
`backend/app/api/v1/system.py`

#### API端点

**GET /api/v1/system/refresh-settings**
- 功能：获取刷新设置
- 返回：settings + modules配置

**POST /api/v1/system/refresh-settings**
- 功能：保存刷新设置
- 参数：settings + modules数组
- 返回：保存结果

## 最佳实践

### 推荐配置

#### 高实时性需求
```
Dashboard 价格数据: 1秒
点差数据表: 1秒
策略面板: 1秒
其他模块: 5-30秒
启用: WebSocket + 页面可见性检测
```

#### 平衡配置（推荐）
```
关键数据: 1-2秒
监控数据: 5秒
静态数据: 10-30秒
启用: 页面可见性检测 + 批量请求
```

#### 低负载配置
```
所有模块: 5秒以上
启用: 所有优化选项
不可见倍率: 10x
```

### 优化建议

1. **关键交易数据**
   - 保持1秒高频刷新
   - 确保实时性

2. **监控数据**
   - 使用5秒中频刷新
   - 平衡性能和实时性

3. **静态数据**
   - 使用30秒低频刷新
   - 减少不必要的请求

4. **页面不可见**
   - 自动降低刷新频率
   - 节省资源

5. **WebSocket**
   - 对于1秒刷新的数据
   - 考虑使用WebSocket推送

## 注意事项

1. **设置生效**
   - 保存后立即生效
   - 通过事件广播通知各模块
   - 需要各模块监听事件并响应

2. **默认值**
   - 点击"恢复默认值"重置所有配置
   - 默认值基于最佳实践

3. **性能影响**
   - 过高的刷新频率会增加负载
   - 建议根据实际需求调整
   - 关注性能建议提示

4. **WebSocket**
   - 需要后端支持
   - 当前为开关，实际功能需要实现
   - 建议优先实现高频数据的WebSocket
