## 一、系统概述

### 1.1 策略类型
系统支持两种套利策略类型：
- **正向套利** (Forward Arbitrage): Binance做多 + Bybit做空
- **反向套利** (Reverse Arbitrage): Bybit做多 + Binance做空

### 1.2 核心功能
- 阶梯式挂单配置（最多5级阶梯）
- 实时点差监控和显示
- 触发次数控制机制
- 开仓/平仓独立配置
- 批量下单和追单机制

## 三、阶梯配置详情

### 3.1 阶梯配置结构

每个阶梯包含以下字段：
```javascript
{
  enabled: Boolean,           // 是否启用该阶梯
  openPrice: Number,          // 开仓点差值
  threshold: Number,          // 平仓点差值
  totalQty: Number,           // 总手数
  openingTriggerCount: Number,  // 开仓触发次数
  closingTriggerCount: Number   // 平仓触发次数
}
```

### 3.2 阶梯执行逻辑

**顺序执行规则**:
1. 阶梯按顺序执行（阶梯1 → 阶梯2 → 阶梯3...）
2. 前一阶梯完成后才启动下一阶梯
3. 每个阶梯独立配置开仓/平仓参数
4. 最多支持5级阶梯

**触发机制**:
- 点差满足条件时触发次数+1
- 累计触发次数达到配置值后执行下单
- 下单完成后重置触发次数

## 四、点差计算公式（最新版）

### 4.1 反向套利

**反向开仓** (Bybit做多):
```
bybit做多点差 = binance_ask - bybit_ask
触发条件: bybit做多点差 >= 反向开仓点差值
```

**反向平仓**:
```
bybit平仓点差 = binance_bid - bybit_bid
触发条件: bybit平仓点差 <= 反向平仓点差值
```

### 4.2 正向套利

**正向开仓** (Binance做多):
```
binance做多点差 = bybit_bid - binance_bid
触发条件: binance做多点差 >= 正向开仓点差值
```

**正向平仓**:
```
binance平仓点差 = bybit_ask - binance_ask
触发条件: binance平仓点差 <= 正向平仓点差值
```

## 七、订单执行流程

### 7.1 当前实现

**批量开仓流程**:
1. 用户点击"批量开仓"按钮
2. 系统检查阶梯配置
   - 验证至少有一个阶梯已启用
   - 检查阶梯参数完整性（开仓点差值、总手数）
   - 验证单次下单手数配置
3. 按阶梯顺序执行
   - 从阶梯1开始顺序执行
   - 前一阶梯完成后才启动下一阶梯
4. Binance限价单 → Bybit市价单
   - Binance先挂限价单
   - 等待Binance成交
   - Bybit立即下市价单
5. 追单机制（最多3次重试）
   - 检测Bybit成交情况
   - 未完全成交则追单

**批量平仓流程**:
1. 用户点击"批量平仓"按钮
2. **系统检查阶梯配置**（新增）
   - 验证至少有一个阶梯已启用
   - 检查阶梯参数完整性（平仓点差值、总手数）
   - 验证单次下单手数配置
   - 检查平仓触发次数配置
3. 系统检查持仓情况
   - 查询当前持仓数量
   - 验证持仓是否足够平仓
   - 计算可平仓数量
4. 按阶梯顺序平仓
   - 从阶梯1开始顺序执行
   - 前一阶梯完成后才启动下一阶梯
   - 累计平仓数量不超过持仓总数
5. Binance限价单 → Bybit市价单
   - Binance先挂限价单（平仓方向）
   - 等待Binance成交
   - Bybit立即下市价单（平仓方向）
6. 追单机制
   - 检测Bybit成交情况
   - 未完全成交则追单
7. 更新持仓记录
   - 记录平仓数量
   - 更新剩余持仓

### 7.2 新版策略执行流程（基于V2.0修正版）

#### 7.2.1 反向开仓完整流程

```
用户点击"反向开仓"按钮
    ↓
【步骤1】系统初始化
    - 触发次数计数器 = 0
    - 当前阶梯索引 = 0
    - 当前持仓总数 = 0
    ↓
【步骤2】检查阶梯配置
    - 验证阶梯1已启用
    - 检查开仓点差值配置
    - 检查总手数配置
    - 检查触发次数配置
    ↓
【步骤3】触发次数检测循环
    while 触发次数 < 配置的触发次数:
        计算 bybit做多点差 = binance_ask - bybit_ask
        if 点差 >= 反向开仓点差值:
            触发次数 += 1
            等待0.1秒（防止重复触发）
    ↓
【步骤4】计算本次下单数量
    剩余数量 = 阶梯总手数 - 当前持仓
    下单数量 = min(开仓单次下单手数, 剩余数量)
    ↓
【步骤5】再次检查点差
    if 当前点差 < 反向开仓点差值:
        重置触发次数
        返回步骤3
    ↓
【步骤6】执行双边下单
    6.1 Binance限价卖单（开空）
        - 价格: binance_bid（买一价）
        - 数量: 下单数量
        - 订单类型: LIMIT

    6.2 监控Binance订单（0.2秒超时）
        情况1: 完全成交 → 继续步骤6.3
        情况2: 0.2秒未完全成交 → 撤单 → 使用已成交数量继续

    6.3 Bybit市价买单（开多）
        - 价格: 市价
        - 数量: Binance实际成交数量
        - 订单类型: MARKET

    6.4 监控Bybit订单（0.1秒后检测）
        if 未完全成交:
            撤单
            按差额重新下单（循环一次）
    ↓
【步骤7】更新持仓
    当前持仓 += min(Binance成交量, Bybit成交量)
    ↓
【步骤8】检查是否继续
    if 当前持仓 >= 阶梯总手数:
        当前阶梯完成
        移动到下一阶梯
        重置当前持仓 = 0
        if 所有阶梯完成:
            结束
        else:
            返回步骤2（检查下一阶梯配置）
    else:
        重置触发次数 = 0
        返回步骤3（继续当前阶梯）
```

#### 7.2.2 反向平仓完整流程

```
用户点击"反向平仓"按钮
    ↓
【步骤1】系统初始化
    - 触发次数计数器 = 0
    - 当前阶梯索引 = 0
    - 累计平仓数量 = 0
    ↓
【步骤2】检查阶梯配置（新增）
    - 验证阶梯1已启用
    - 检查平仓点差值配置
    - 检查总手数配置
    - 检查触发次数配置
    - 验证配置参数合法性
    ↓
【步骤3】检查持仓情况
    - 查询当前反向持仓总数
    - 验证持仓 > 0
    - 计算可平仓数量 = min(阶梯总手数, 剩余持仓)
    ↓
【步骤4】触发次数检测循环
    while 触发次数 < 配置的触发次数:
        计算 bybit平仓点差 = binance_bid - bybit_bid
        if 点差 <= 反向平仓点差值:
            触发次数 += 1
            等待0.1秒（防止重复触发）
    ↓
【步骤5】计算本次下单数量
    剩余可平 = min(阶梯总手数 - 累计平仓, 当前持仓)
    下单数量 = min(平仓单次下单手数, 剩余可平)
    ↓
【步骤6】再次检查点差
    if 当前点差 > 反向平仓点差值:
        重置触发次数
        返回步骤4
    ↓
【步骤7】执行双边平仓
    7.1 Binance限价买单（平仓）
        - 价格: binance_ask（卖一价）
        - 数量: 下单数量
        - 订单类型: LIMIT

    7.2 监控Binance订单（0.2秒超时）
        情况1: 完全成交 → 继续步骤7.3
        情况2: 0.2秒未完全成交 → 撤单 → 使用已成交数量继续

    7.3 Bybit市价卖单（平仓）
        - 价格: 市价
        - 数量: Binance实际成交数量
        - 订单类型: MARKET
        - 标记: reduce_only=True

    7.4 监控Bybit订单（0.1秒后检测）
        if 未完全成交:
            撤单
            按差额重新下单（循环一次）
    ↓
【步骤8】更新持仓
    累计平仓 += min(Binance成交量, Bybit成交量)
    当前持仓 -= min(Binance成交量, Bybit成交量)
    ↓
【步骤9】检查是否继续
    if 累计平仓 >= 阶梯总手数:
        当前阶梯完成
        移动到下一阶梯
        重置累计平仓 = 0
        if 所有阶梯完成 OR 当前持仓 = 0:
            结束
        else:
            返回步骤2（检查下一阶梯配置）
    else:
        重置触发次数 = 0
        返回步骤4（继续当前阶梯）
```

### 7.4 错误处理和异常情况

#### 7.4.1 配置检查失败

**场景**: 阶梯配置不完整或不合法

**处理**:
```javascript
// 前端检查示例
function validateLadderConfig(ladders, action) {
  const errors = []

  // 检查是否有启用的阶梯
  const enabledLadders = ladders.filter(l => l.enabled)
  if (enabledLadders.length === 0) {
    errors.push('至少需要启用一个阶梯')
    return { valid: false, errors }
  }

  // 检查每个启用的阶梯
  enabledLadders.forEach((ladder, index) => {
    if (action === 'opening') {
      if (!ladder.openPrice || ladder.openPrice <= 0) {
        errors.push(`阶梯${index + 1}: 开仓点差值必须大于0`)
      }
      if (!ladder.openingTriggerCount || ladder.openingTriggerCount < 1) {
        errors.push(`阶梯${index + 1}: 开仓触发次数必须至少为1`)
      }
    } else {
      if (!ladder.threshold || ladder.threshold <= 0) {
        errors.push(`阶梯${index + 1}: 平仓点差值必须大于0`)
      }
      if (!ladder.closingTriggerCount || ladder.closingTriggerCount < 1) {
        errors.push(`阶梯${index + 1}: 平仓触发次数必须至少为1`)
      }
    }

    if (!ladder.totalQty || ladder.totalQty <= 0) {
      errors.push(`阶梯${index + 1}: 总手数必须大于0`)
    }
  })

  return {
    valid: errors.length === 0,
    errors
  }
}
```

**用户提示**:
- 显示具体错误信息
- 高亮错误配置项
- 提供修复建议

#### 7.4.3 订单执行失败

**处理**:

#### 7.4.4 网络异常

**场景**: 网络连接中断或超时

**处理**:

3. 保存当前执行状态
