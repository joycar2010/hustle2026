# 通知模板声音设置调试指南

## 问题描述
在通知模板管理中编辑模板设置声音后，保存时声音设置没有保存成功。

## 调试步骤

### 步骤1: 检查浏览器控制台
1. 打开 http://13.115.21.77:3000/system
2. 按 F12 打开开发者工具
3. 切换到 "Console" 标签
4. 清空控制台（点击🚫图标）

### 步骤2: 测试编辑功能
1. 点击"总资产提醒"模板的"编辑"按钮
2. 查看控制台是否输出：
   ```
   编辑模板: {...}
     - alert_sound值: ...
   ```
3. 如果没有看到这些日志，说明前端代码没有正确加载

### 步骤3: 设置声音
1. 在"声音提醒设置"部分，从下拉框选择一个声音文件（如 spread.mp3）
2. 设置重复次数（如 5）
3. 点击"保存"按钮

### 步骤4: 检查保存日志
查看控制台是否输出以下日志：
```
保存模板 - 完整数据: {...}
alert_sound: spread.mp3
repeat_count: 5
保存响应: {...}
加载的模板数量: 20
第一个模板示例: {...}
  - alert_sound: ...
  - repeat_count: ...
```

### 步骤5: 检查Network标签
1. 切换到 "Network" 标签
2. 筛选 "Fetch/XHR"
3. 再次保存模板
4. 查看是否有 PUT 请求到 `/api/v1/notifications/templates/...`
5. 点击该请求，查看：
   - Request Payload（发送的数据）
   - Response（服务器响应）

## 可能的问题和解决方案

### 问题1: 控制台没有任何日志
**原因**: 前端代码没有正确加载
**解决**:
1. 硬刷新页面（Ctrl + Shift + R 或 Ctrl + F5）
2. 清除浏览器缓存
3. 重启前端服务

### 问题2: 有"编辑模板"日志，但没有"保存模板"日志
**原因**: 保存按钮点击事件没有触发
**解决**:
1. 检查是否有JavaScript错误
2. 检查Console标签是否有红色错误信息

### 问题3: 有"保存模板"日志，但alert_sound是空字符串或null
**原因**: 下拉框没有正确选择
**解决**:
1. 确认下拉框中有声音文件选项
2. 确认选择了非空选项（不是"不使用声音"）

### 问题4: Network标签没有PUT请求
**原因**: API调用被阻止或失败
**解决**:
1. 检查Console是否有CORS错误
2. 检查Network标签是否有失败的请求（红色）

### 问题5: PUT请求返回错误
**原因**: 后端处理失败
**解决**:
1. 查看Response内容
2. 检查后端日志：`tail -100 /tmp/backend.log`

## 手动测试数据库

如果以上步骤都正常，但数据还是没保存，可以手动测试数据库：

```bash
cd /c/app/hustle2026/backend
python scripts/test_sound_setting.py
```

这个脚本会直接测试数据库的保存和读取功能。
