# 交易统计优化说明

## 优化内容

### 1. 详细日志记录 ✅

**功能**：记录每笔交易的盈亏计算过程

**实现细节**：
- 在 `_build_stats()` 函数中添加 `enable_logging` 参数
- 记录每笔交易的开仓/平仓操作
- 记录仓位变化和盈亏计算详情
- 记录最终统计汇总和剩余持仓

**日志示例**：
```
[Binance] 做多买入 XAUUSDT: qty=0.1, price=2650.50, 仓位 0.0000 -> 0.1000
[Binance] 做多平仓 XAUUSDT: 平仓qty=0.1, 开仓均价=2650.50, 平仓价=2655.00, 盈亏=0.43 USDT, 仓位 0.1000 -> 0.0000
```

### 2. 做空支持 ✅

**功能**：扩展FIFO计算支持做空场景

**实现逻辑**：
- 仓位数量 `qty > 0`：做多仓位
- 仓位数量 `qty < 0`：做空仓位
- 成本 `cost`：正值表示做多成本，负值表示做空成本

**做多场景**：
- 买入：增加多头仓位
- 卖出：平掉多头仓位，计算盈亏 = (平仓价 - 开仓价) × 数量 - 手续费

**做空场景**：
- 卖出：增加空头仓位
- 买入：平掉空头仓位，计算盈亏 = (开仓价 - 平仓价) × 数量 - 手续费

**混合场景**：
- 多头平仓后如有剩余卖单，自动转为做空
- 空头平仓后如有剩余买单，自动转为做多

### 3. 性能优化 ✅

**功能**：大量数据时使用分批处理

**优化策略**：
- 批次大小：5000 笔订单/批次
- 自动检测：订单数量 > 5000 时启用分批处理
- 日志控制：大数据集关闭详细日志，提升性能
- 内存优化：分批加载避免一次性加载过多数据

**性能对比**：
- 小数据集 (< 5000)：启用详细日志，单次加载
- 大数据集 (≥ 5000)：关闭详细日志，分批加载

## 技术细节

### FIFO 仓位跟踪结构

```python
position = {
    'qty': float,   # 仓位数量（正=做多，负=做空）
    'cost': float   # 成本（正=做多成本，负=做空成本）
}
```

### 盈亏计算公式

**做多盈亏**：
```
realized_pnl = (平仓价 - 开仓均价) × 平仓数量 - 手续费
```

**做空盈亏**：
```
realized_pnl = (开仓均价 - 平仓价) × 平仓数量 - 手续费
```

## 注意事项

1. **日志级别**：生产环境建议设置为 INFO 级别
2. **性能影响**：详细日志会增加 I/O 开销，大数据集自动关闭
3. **数据顺序**：必须按时间升序处理订单，确保 FIFO 准确性
4. **混合持仓**：系统自动处理多空转换，无需手动干预
