## 账户卡片结构

每个账户卡片分为两个主要部分：

### 1. 卡片头部 (Header)
```
┌─────────────────────────────────────────────────────────┐
│ ● 账户名称                              [连接] [禁用]   │
│   平台名称 (Binance/Bybit MT5)                         │
│   状态 (已激活/未连接/连接失败)                        │
│                                                          │
│   错误信息 (如果有)                                     │
└─────────────────────────────────────────────────────────┘
```

**组成元素：**
- 状态指示灯（绿色/红色/黄色圆点）
- 账户名称（金色加粗）
- 平台标识（Binance / Bybit MT5）
- 连接状态文本
- 操作按钮：
  - "连接"/"断开" 按钮（绿色/红色）
  - "禁用" 按钮（灰色，需先断开连接）
- 错误提示框（黄色背景，显示错误信息或限制时间）

### 2. 账户数据区域 (Data Section)

#### 通用字段（所有账户）
```
账户总资产        XXX.XX USDT
可用总资产        XXX.XX USDT
净资产            XXX.XX USDT
总持仓            X.XX 手          ← 单位为"手"
冻结资产          XXX.XX USDT
当日盈亏          +/-XXX.XX USDT   ← 带颜色（绿色/红色）
保证金余额        XXX.XX USDT
风险率            XX.XX%           ← 带颜色（绿/黄/红）
```

#### Bybit MT5 特有字段 (platform_id === 2)
```
手续费(佣金)      +/-XXX.XX USDT
MT5过夜费         +/-XXX.XX USDT
```

#### Binance 特有字段 (platform_id === 1)
```
手续费汇总(USDT)  +/-XXX.XX USDT
手续费汇总(BNB)   +/-XXX.XX USDT
```

## 响应式设计

### 移动端 (Mobile)
- 内边距：`p-3`
- 账户名称和状态垂直堆叠
- 按钮紧凑排列

### 桌面端 (Desktop)
- 内边距：`p-4`
- 更大的间距：`space-y-3`
- 更宽松的布局

## 数据显示规则

### 1. 数值格式化
- **USDT金额**：`XXX,XXX.XX USDT`（千位分隔符，2位小数）
- **手数**：`X.XX 手`（2位小数）
- **百分比**：`XX.XX%`（2位小数）
- **带符号**：`+XXX.XX` 或 `-XXX.XX`（盈亏类字段）

### 2. 颜色编码
- **盈亏颜色**：
  - 正值：绿色 `text-[#0ecb81]`
  - 负值：红色 `text-[#f6465d]`

- **风险率颜色**：
  - < 30%：绿色 `text-[#0ecb81]`
  - 30-60%：黄色 `text-[#f0b90b]`
  - > 60%：红色 `text-[#f6465d]`

- **状态指示灯**：
  - 正常：绿色 `bg-[#0ecb81]`
  - 错误：黄色 `bg-[#f0b90b]`
  - 未连接：红色 `bg-[#f6465d]`

### 3. 特殊状态显示
- **连接失败**：显示"暂无"或错误信息
- **限流封禁**：显示解封时间（如：`限制至3:45 PM (15分钟)`）
- **无数据**：显示"暂无"

## 交互功能

### 1. 连接管理
- **断开连接**：点击"断开"按钮，断开WebSocket连接
- **重新连接**：点击"连接"按钮，有10秒冷却时间防止Binance限制
- **禁用账户**：必须先断开连接，然后才能禁用

### 2. 状态持久化
- 断开状态保存在 `localStorage`
- 重连冷却时间记录在 `localStorage`
- WebSocket连接状态保存在 `localStorage`

### 3. 实时更新
- 通过WebSocket接收 `account_balance` 消息
- 自动更新账户余额和持仓数据
- 更新系统提醒（通过notification store）

```
│                                                          │

```

## 字段映射关系

### 后端数据 → 前端显示

| 后端字段 | 前端显示 | 单位 | 平台 |
|---------|---------|------|------|
| `total_assets` | 账户总资产 | USDT | 全部 |
| `available_balance` | 可用总资产 | USDT | 全部 |
| `net_assets` | 净资产 | USDT | 全部 |
| `total_positions` | 总持仓 | 手 | 全部 |
| `frozen_assets` | 冻结资产 | USDT | 全部 |
| `daily_pnl` | 当日盈亏 | USDT | 全部 |
| `margin_balance` | 保证金余额 | USDT | 全部 |
| `risk_ratio` | 风险率 | % | 全部 |
| `commission_fee` | 手续费(佣金) | USDT | Bybit MT5 |
| `funding_fee` | MT5过夜费 | USDT | Bybit MT5 |
| `long_funding_rate` | 手续费汇总(USDT) | USDT | Binance |
| `short_funding_rate` | 手续费汇总(BNB) | USDT | Binance |

## 技术实现

### 组件依赖
- `useMarketStore`：WebSocket连接和消息监听
- `useNotificationStore`：系统提醒更新
- `api`：REST API调用

### 关键方法
- `fetchAccountData()`：获取账户数据
- `handleAccountBalanceUpdate()`：处理WebSocket更新
- `toggleConnection()`：切换连接状态
- `toggleDisable()`：禁用账户
- `getDisplayValue()`：格式化显示值
- `getValueColor()`：获取数值颜色
- `getRiskColor()`：获取风险率颜色

### WebSocket集成
- 监听 `account_balance` 消息类型
- 后端每10秒广播一次
- 自动更新账户数据和系统提醒

## 样式特点

- **背景色**：深色主题 `bg-[#252930]`
- **边框色**：`border-[#2b3139]`
- **文字色**：
  - 标签：灰色 `text-gray-400`
  - 数值：白色（默认）
  - 强调：金色 `text-[#D4B106]`
- **圆角**：`rounded-lg`
- **间距**：`space-y-2` (移动端) / `space-y-3` (桌面端)
- **字体**：数值使用等宽字体 `font-mono`
