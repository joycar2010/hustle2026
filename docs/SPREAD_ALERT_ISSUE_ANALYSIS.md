# 飞书点差消息推送问题分析

## 问题描述
用户cq987和admin设置的风险阈值不一样，但收到的飞书消息却是一样的。

## 数据库查询结果

| 用户名 | 正向开仓阈值 | 正向平仓阈值 | 反向开仓阈值 | 反向平仓阈值 |
|--------|------------|------------|------------|------------|
| admin  | 4.0        | 1.0        | 5.0        | 1.0        |
| cq987  | 5.0        | 1.0        | 3.0        | -5.0       |

## 问题分析

### 1. 配置差异
两个用户的阈值配置确实不同：
- **admin**: 正向开仓4.0, 反向开仓5.0
- **cq987**: 正向开仓5.0, 反向开仓3.0

### 2. 异常配置
**cq987用户的反向平仓阈值为-5.0，这是一个异常值！**

正常情况下：
- 开仓阈值应该 > 0（点差达到一定值时开仓）
- 平仓阈值应该 > 0 且 < 开仓阈值（点差回落时平仓）

### 3. 可能的问题原因

#### 原因1：消息模板问题
检查飞书消息模板是否使用了固定的阈值，而不是用户的实际阈值。

#### 原因2：消息内容相同
虽然触发条件不同，但消息内容可能没有包含用户的阈值信息，导致看起来一样。

#### 原因3：冷却时间导致
如果两个用户在短时间内触发，可能因为冷却时间机制，实际上只有一个用户收到了消息。

#### 原因4：WebSocket推送问题
如果使用WebSocket推送，可能存在广播问题，导致所有用户收到相同的消息。

## 验证步骤

### 步骤1：检查消息模板
查看飞书消息模板是否包含用户阈值信息。

### 步骤2：检查消息发送日志
查看notification_logs表，确认两个用户是否都收到了消息，以及消息内容是否不同。

### 步骤3：检查当前市场点差
查看当前的正向和反向点差值，判断哪些用户应该收到消息。

### 步骤4：检查代码逻辑
确认spread_alert_service.py中的逻辑是否正确使用了用户的阈值。

## 建议修复

### 1. 修复异常阈值
cq987用户的反向平仓阈值-5.0需要修正为正常值（如0.5或1.0）。

### 2. 增强消息内容
在飞书消息中明确显示：
- 用户配置的阈值
- 当前实际点差值
- 触发的具体条件

### 3. 添加调试日志
在spread_alert_service.py中添加详细日志，记录：
- 每个用户的阈值配置
- 点差比较结果
- 是否触发发送

### 4. 前端验证
在前端风险设置页面添加输入验证：
- 阈值必须 > 0
- 平仓阈值 < 开仓阈值
- 提供合理的默认值
