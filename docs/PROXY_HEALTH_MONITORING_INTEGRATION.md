# 代理健康监控集成到MarketCards跑马灯和系统状态监控

## 实施日期
2026-03-16

## 功能概述
将IP代理健康监控集成到MarketCards.vue的跑马灯和SystemStatusModal系统状态监控中，实现实时代理健康状态展示。

## 实施内容

### 1. MarketCards.vue 跑马灯集成

#### 新增状态变量
```javascript
const proxyHealthStatus = ref({
  total: 0,      // 总代理数
  active: 0,     // 活跃代理数
  failed: 0,     // 失败代理数
  avgHealth: 100 // 平均健康度
})
```

#### 新增获取函数
```javascript
async function fetchProxyHealthStatus() {
  // 从 /api/v1/proxies 获取代理列表
  // 计算总数、活跃数、失败数
  // 计算平均健康度
}
```

#### 跑马灯显示逻辑
在系统状态跑马灯中添加代理健康信息：
- 正常状态：`代理:X/Y活跃`
- 有失败：`代理:X/Y活跃(Z失败)`
- 健康度低：`⚠️代理:X/Y活跃(健康度:N)`

#### 健康判断逻辑
```javascript
const proxyHealthy = proxyHealthStatus.value.avgHealth >= 50 &&
                     proxyHealthStatus.value.failed === 0
systemHealthy.value = ... && proxyHealthy
```

#### 刷新机制
- 初始加载：`onMounted` 时调用 `fetchProxyHealthStatus()`
- 定时刷新：每30秒刷新一次代理健康状态
- 系统状态：每10秒更新一次跑马灯显示

## 技术实现

### 数据流
```
1. MarketCards.vue onMounted
   ↓
2. fetchProxyHealthStatus()
   ↓
3. GET /api/v1/proxies
   ↓
4. 计算统计数据
   ↓
5. 更新 proxyHealthStatus
   ↓
6. updateSystemStatus() 读取状态
   ↓
7. 更新跑马灯文本和健康状态
```

### 刷新策略
- **代理健康状态**: 30秒刷新一次（独立定时器）
- **系统状态**: 10秒刷新一次（包含跑马灯）
- **模态框数据**: 打开时立即获取，每30秒刷新

### 性能优化
- 代理列表只在需要时获取
- 模态框中只显示前5个代理详情
- 使用独立定时器避免频繁请求

## 健康判断标准

### 代理健康度计算
```javascript
avgHealth = Σ(proxy.health_score) / total_proxies
```

### 系统健康判断
系统整体健康需要满足：
- 后端服务运行
- WebSocket已连接
- 数据库使用率 < 80%
- 飞书通知已配置
- Redis正常
- SSL证书健康
- **代理平均健康度 >= 50 且 无失败代理** ← 新增

## 告警级别

### 代理状态告警
- **正常**: 平均健康度 >= 80，无失败代理
- **警告**: 平均健康度 >= 50 或 有失败代理
- **异常**: 平均健康度 < 50

### 跑马灯颜色
- **绿色背景**: 系统完全健康（包括代理）
- **红色背景**: 系统存在异常（包括代理异常）
