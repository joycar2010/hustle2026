# 飞书通知逻辑汇总

## 一、飞书服务架构

### 1.1 核心服务类
**文件位置**: `backend/app/services/feishu_service.py`

**主要功能**:
- **FeishuService类**: 飞书机器人通知服务的核心实现
- **认证机制**: 使用app_id和app_secret获取tenant_access_token
- **Token管理**: 自动刷新token（有效期2小时，提前5分钟刷新）
- **时区处理**: 使用北京时间（UTC+8）

### 1.2 消息类型

#### 1.2.1 文本消息
```python
send_text_message(receive_id, content, receive_id_type="open_id")
```
- 发送纯文本消息
- 支持多种接收者ID类型：open_id, user_id, union_id, email, chat_id

#### 1.2.2 卡片消息
```python
send_card_message(receive_id, title, content, receive_id_type="open_id", color="blue")
```
- 发送美观的卡片消息
- 支持颜色主题：blue(信息)、green(成功)、orange(警告)、red(危险)、grey(系统)、turquoise(特殊)
- 使用深色主题风格，与系统UI一致

#### 1.2.3 带音频的卡片消息
```python
send_card_with_audio(receive_id, title, content, audio_url, receive_id_type="open_id", color="blue")
```
- 在卡片消息中嵌入音频播放器
- 支持自定义提醒音频

## 二、通知触发场景

### 2.1 点差提醒服务
**文件位置**: `backend/app/services/spread_alert_service.py`

**触发条件**:
1. **正向套利提醒**:
   - 正开差：当Binance-Bybit点差 ≥ 设定的正开差阈值
   - 正平差：当Binance-Bybit点差 ≤ 设定的正平差阈值
   - 同步条数：连续满足条件达到设定的同步条数

2. **反向套利提醒**:
   - 反开差：当Bybit-Binance点差 ≥ 设定的反开差阈值
   - 反平差：当Bybit-Binance点差 ≤ 设定的反平差阈值
   - 同步条数：连续满足条件达到设定的同步条数

**通知内容**:
- 提醒类型（正向开仓/平仓、反向开仓/平仓）
- 当前点差值
- 阈值设置
- 连续满足条数
- 时间戳

**颜色规则**:
- 开仓提醒：green（绿色）
- 平仓提醒：orange（橙色）

### 2.2 风险提醒服务
**文件位置**: `backend/app/services/risk_alert_service.py`

**触发条件**:
1. **单腿风险提醒**:
   - 当Binance或Bybit单边持仓超过设定的单腿阈值
   - 检测到持仓不平衡

2. **净资产提醒**:
   - 当账户净资产低于设定的净资产阈值
   - 账户资金出现异常波动

3. **MT5连接状态提醒**:
   - MT5连接断开
   - MT5连接恢复

**通知内容**:
- 风险类型
- 当前值和阈值
- 风险等级
- 建议操作
- 时间戳

**颜色规则**:
- 严重风险：red（红色）
- 警告：orange（橙色）
- 恢复正常：green（绿色）

### 2.3 手动通知测试
**文件位置**: `backend/app/api/v1/notifications.py`

**功能**:
- 管理员可以手动发送测试通知
- 支持自定义标题、内容、颜色
- 支持带音频的通知测试
- 用于验证飞书服务配置是否正常

## 三、通知配置管理

### 3.1 数据库配置
**表**: `notification_configs`

**字段**:
- `service_type`: 服务类型（feishu）
- `is_enabled`: 是否启用
- `config_data`: JSON配置数据
  - `app_id`: 飞书应用ID
  - `app_secret`: 飞书应用密钥

### 3.2 用户配置
**表**: `users`

**字段**:
- `feishu_open_id`: 用户的飞书open_id
- `feishu_enabled`: 是否启用飞书通知（用户级别开关）

### 3.3 提醒阈值配置
**表**: `alert_settings`

**字段**:
- `reverse_open_price`: 反开差阈值
- `reverse_close_price`: 反平差阈值
- `reverse_open_sync_count`: 反开步（同步条数）
- `reverse_close_sync_count`: 反平步（同步条数）
- `forward_open_price`: 正开差阈值
- `forward_close_price`: 正平差阈值
- `forward_open_sync_count`: 正开步（同步条数）
- `forward_close_sync_count`: 正平步（同步条数）
- `single_leg_threshold`: 单腿提醒阈值
- `net_asset_threshold`: 净资产提醒阈值

**注意**:
- 所有阈值字段**允许为空**（null）
- 当字段为空时，对应的提醒功能将不会触发
- 用户可以根据需要选择性启用某些提醒

## 四、通知发送流程

### 4.1 初始化流程
1. 系统启动时从数据库读取飞书配置
2. 如果配置存在且启用，初始化FeishuService实例
3. 获取并缓存tenant_access_token

### 4.2 提醒检测流程
1. **点差监控**:
   - WebSocket实时接收市场数据
   - spread_alert_service持续监控点差变化
   - 满足条件时触发通知

2. **风险监控**:
   - 定期检查账户持仓和资产状态
   - risk_alert_service检测风险指标
   - 超过阈值时触发通知

### 4.3 消息发送流程
1. 检查飞书服务是否已初始化
2. 检查用户是否启用飞书通知
3. 获取用户的feishu_open_id
4. 构造消息内容（标题、正文、颜色）
5. 调用飞书API发送消息
6. 记录发送结果日志

### 4.4 错误处理
- Token过期自动刷新
- 发送失败记录错误日志
- 不影响系统主要功能运行

## 六、安全与隐私

### 6.1 敏感词规避
- 使用"生鲜商品配送"等中性词汇
- 避免直接使用金融交易相关敏感词

### 6.2 数据安全
- app_secret加密存储在数据库
- token仅在内存中缓存
- 不记录用户消息内容

### 6.3 权限控制
- 只有管理员可以配置飞书服务
- 用户只能管理自己的通知设置
- 通知仅发送给授权用户

## 七、监控与维护

### 7.1 状态监控
- 系统状态监控模态框显示飞书服务状态
- 实时显示连接状态和配置状态
- 30秒自动刷新

### 7.2 日志记录
- 记录所有消息发送操作
- 记录token刷新操作
- 记录错误和异常

### 7.3 故障排查
1. 检查飞书服务配置是否正确
2. 检查用户feishu_open_id是否配置
3. 检查网络连接是否正常
4. 查看后端日志获取详细错误信息

## 九、最佳实践建议

### 9.1 阈值设置
- 根据市场波动情况合理设置点差阈值
- 单腿阈值建议设置为正常持仓的20-30%
- 净资产阈值建议设置为初始资金的80%

### 9.2 通知频率控制
- 避免设置过低的阈值导致频繁通知
- 使用同步条数过滤短期波动
- 重要提醒可以配置音频增强提示

### 9.3 测试验证
- 配置完成后先发送测试通知
- 验证用户能正常接收消息
- 定期检查飞书服务状态

## 十、技术细节

### 10.1 API端点
- 获取token: `https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal`
- 发送消息: `https://open.feishu.cn/open-apis/im/v1/messages`

### 10.2 消息格式
- 文本消息: `msg_type: "text"`
- 卡片消息: `msg_type: "interactive"`
- 支持Markdown格式

### 10.3 性能优化
- Token缓存减少API调用
- 异步发送不阻塞主流程
- 批量通知合并发送
