# 下单数量转换修复说明

## 修复日期
2026-02-22

## 问题描述

### 原问题
系统在Binance和Bybit之间下单时，使用相同的数量值，但两个交易所的合约单位不同：

- **Binance XAUUSDT (Perpetual)**: 1 Contract = 1 XAU (盎司)
- **Bybit XAUUSD.s (TradFi)**: 1 Lot = 100 XAU (盎司)

这导致如果Binance下单1手（1盎司），Bybit也下单1手（100盎司），数量相差100倍！

### 影响
- 资金使用不匹配
- 套利比例严重失衡
- 可能导致爆仓风险

---

## 修复内容

### 1. 后端修复 - order_executor.py

#### 修改位置
文件：`backend/app/services/order_executor.py`

#### 核心修改

**1.1 添加bybit_quantity参数**
```python
async def execute_dual_order(
    self,
    # ... 其他参数
    quantity: float,  # Binance数量（合约）
    bybit_quantity: Optional[float] = None,  # Bybit数量（手）
) -> Dict[str, Any]:
    """
    Note:
        Binance XAUUSDT: 1 contract = 1 XAU (ounce)
        Bybit XAUUSD.s: 1 lot = 100 XAU (ounces)
        Therefore: bybit_quantity = binance_quantity / 100
    """
```

**1.2 自动计算Bybit数量**
```python
# 第395-398行
# Calculate Bybit quantity if not provided
# Binance XAUUSDT: 1 contract = 1 XAU; Bybit XAUUSD.s: 1 lot = 100 XAU
if bybit_quantity is None:
    bybit_quantity = quantity / 100.0
```

**1.3 精度处理**
```python
# 第400-404行
# Round quantities to correct precision
# Binance XAUUSDT: min 0.001, step 0.001 → 3 decimal places
quantity = round(quantity, 3)
# Bybit MT5 XAUUSD.s: min 0.01 lot, step 0.01 → 2 decimal places
bybit_quantity = round(bybit_quantity, 2)
```

**1.4 最小数量验证**
```python
# 第406-419行
# Binance XAUUSDT: minimum 0.001 XAU
if quantity < 0.001:
    return {"success": False, "error": "Binance数量不足: {quantity} XAU < 0.001 XAU"}

# Bybit MT5 XAUUSD.s: minimum 0.01 Lot
if bybit_quantity < 0.01:
    return {"success": False, "error": "Bybit数量不足: {bybit_quantity} Lot < 0.01 Lot"}
```

**1.5 下单时使用转换后的数量**
```python
# 第450-468行
binance_result, bybit_result = await asyncio.gather(
    self.place_binance_order(
        # ...
        quantity,  # Binance使用原始数量
        # ...
    ),
    self.place_bybit_order(
        # ...
        str(bybit_quantity),  # Bybit使用转换后的数量
        # ...
    ),
)
```

**1.6 追单逻辑中的数量转换**
```python
# 追单时也需要正确转换数量
# 获取已成交数量
binance_filled_qty = binance_status.get("filled_qty", 0)
bybit_filled_qty = bybit_status.get("filled_qty", 0)

# 将Bybit成交数量转换为Binance单位进行比较
bybit_filled_qty_in_binance_units = bybit_filled_qty * 100

# 计算追单数量
if binance_filled and not bybit_filled:
    binance_chase_qty = binance_filled_qty
    bybit_chase_qty = binance_filled_qty / 100.0  # 转换为Bybit单位
elif bybit_filled and not binance_filled:
    binance_chase_qty = bybit_filled_qty_in_binance_units
    bybit_chase_qty = bybit_filled_qty
```

## 注意事项

### 1. 最小交易量限制
- **Binance XAUUSDT**: 最小0.001 XAU (0.001 contract)
- **Bybit XAUUSD.s**: 最小0.01 Lot (1 XAU)

因此，系统最小交易量为 **1 XAU**（Bybit限制）

### 2. 精度要求
- **Binance**: 3位小数 (0.001)
- **Bybit**: 2位小数 (0.01)

系统会自动四舍五入到正确精度。

### 3. 资金要求
假设金价2700 USD/XAU：

**交易1 XAU所需保证金**：
- Binance (20倍杠杆): 2700 / 20 = 135 USDT
- Bybit (100倍杠杆): 2700 / 100 = 27 USDT
- **总计**: ~162 USDT

### 4. 追单逻辑
追单时也会自动转换数量：
- 如果Binance成交10 XAU，追Bybit时会下0.10 Lot
- 如果Bybit成交0.05 Lot (5 XAU)，追Binance时会下5 contracts

## 常见问题

### Q1: 为什么Bybit数量总是Binance的1/100？
**A**: 因为Bybit的1 Lot = 100 XAU，而Binance的1 Contract = 1 XAU。为了保持相同的实际交易量（盎司数），需要除以100。

### Q2: 如果我想交易0.5 XAU怎么办？
**A**: 无法交易，因为Bybit最小交易量是0.01 Lot = 1 XAU。系统会返回错误提示。

### Q3: 追单时数量会正确转换吗？
**A**: 会的。追单逻辑已经更新，会自动在两个交易所之间转换数量单位。

### Q4: 如果我手动指定bybit_quantity会怎样？
**A**: 系统会使用你指定的值，不会自动计算。但不建议这样做，除非你清楚知道自己在做什么。
