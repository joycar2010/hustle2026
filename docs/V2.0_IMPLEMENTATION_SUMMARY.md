### Data Flow

1. **Trigger Detection**:
   - Market data received from exchanges
   - Spread calculated and compared to threshold
   - Trigger added to TriggerCountManager
   - Progress pushed via WebSocket

2. **Execution Decision**:
   - TriggerCountManager checks trigger count
   - If count >= required and within timeout, execute
   - Otherwise, wait for more triggers

3. **Order Execution**:
   - OrderExecutorV2 calculates optimal prices
   - Orders placed on both exchanges simultaneously
   - Retry on failure (max 3 times)
   - Order execution pushed via WebSocket

4. **Position Management**:
   - PositionManager tracks positions
   - Position changes pushed via WebSocket
   - P&L calculated in real-time

5. **Status Updates**:
   - All events queued in StatusPusher
   - Async worker processes queue
   - Events sent to frontend via WebSocket
