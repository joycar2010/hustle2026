# 日志管理功能说明

## 功能概述

在系统管理页面 (http://13.115.21.77:3000/system) 新增"日志管理"标签页，用于查看和管理交易统计计算的详细日志。

## 访问路径

1. 访问 http://13.115.21.77:3000/system
2. 点击顶部导航栏的"日志管理"标签

## 功能特性

### 1. 日志显示
- **实时日志输出**：显示交易统计计算过程的详细日志
- **可配置显示行数**：支持 50/100/200/500 行显示
- **日志分类**：
  - `[Binance]` - Binance平台交易日志
  - `[MT5]` - MT5平台交易日志
- **日志级别**：INFO、WARNING、ERROR
- **颜色标识**：
  - 盈利交易显示为绿色
  - 亏损交易显示为红色
  - 普通日志显示为白色

### 2. 日志内容

日志记录包括：
- **做多买入**：开仓或增加多头仓位
- **做多平仓**：卖出平掉多头仓位，显示盈亏
- **做空卖出**：开仓或增加空头仓位
- **做空平仓**：买入平掉空头仓位，显示盈亏
- **统计汇总**：最终统计结果和剩余持仓

### 3. 控制功能

- **启用/禁用日志**：控制日志记录开关
- **刷新日志**：手动刷新日志内容
- **清空日志**：清除当前显示的所有日志
- **自动刷新**：启用后每5秒自动刷新一次

### 4. 统计信息

- 总日志条数
- 最后更新时间
- 日志状态（运行中/已停止）


## 技术实现

### 前端
- **文件**：`frontend/src/views/System.vue`
- **新增标签页**：日志管理
- **自动刷新**：每5秒调用API获取最新日志
- **响应式设计**：支持不同屏幕尺寸

### 后端
- **文件**：`backend/app/api/v1/system.py`
- **新增端点**：`GET /api/v1/system/logs/trading`
- **日志来源**：从 `app.api.v1.trading` logger 读取
- **返回格式**：JSON格式，包含时间戳、级别、消息

### 日志记录
- **文件**：`backend/app/api/v1/trading.py`
- **Logger**：使用Python标准logging模块
- **日志级别**：INFO
- **触发时机**：每次调用交易统计API时

## 使用场景

1. **调试盈亏计算**：查看每笔交易的详细计算过程
2. **验证FIFO逻辑**：确认仓位跟踪和平仓顺序
3. **排查数据问题**：定位统计数据不准确的原因
4. **监控系统运行**：实时查看交易处理状态

## 注意事项

1. 日志数据存储在内存中，重启后端服务会清空
2. 大量日志可能影响性能，建议定期清空
3. 日志仅记录统计计算过程，不包含实际交易执行
4. 自动刷新功能会增加网络请求，可根据需要关闭
