## Phase 2: Integration Testing

### 2.1 Trigger-to-Execution Flow
**Objective**: Verify complete flow from trigger detection to order execution

**Test Scenarios**:
1. **Scenario 1: Reverse Opening with 3 Triggers**
   - Setup: Configure strategy with 3-trigger requirement
   - Action: Generate 3 valid triggers within timeout
   - Expected: Execute reverse opening orders on both exchanges
   - Verify: Position created, orders executed, WebSocket events sent

2. **Scenario 2: Trigger Timeout**
   - Setup: Configure strategy with 0.2s timeout
   - Action: Generate 2 triggers, wait 0.3s, generate 3rd trigger

3. **Scenario 3: Forward Closing with Position Check**
   - Setup: Create existing positions on both exchanges
   - Action: Trigger forward closing

### 5.2 Deployment Steps
1. **Backup Current System**
   ```bash
   # Backup database
   python backend/scripts/backup_database.py

   # Backup configuration
   cp backend/config/production.yaml backend/config/production.yaml.backup
   ```

2. **Deploy Backend**
   ```bash
   # Stop services
   systemctl stop hustle-backend

   # Update code
   git pull origin main

   # Install dependencies
   pip install -r backend/requirements.txt

   # Run migrations
   alembic upgrade head

   # Start services
   systemctl start hustle-backend
   ```

3. **Deploy Frontend**
   ```bash
   # Build frontend
   cd frontend
   npm run build

   # Deploy to production
   npm run deploy:production
   ```

4. **Verify Deployment**
   ```bash
   # Check service status
   systemctl status hustle-backend

   # Check logs
   tail -f /var/log/hustle/backend.log

   # Run smoke tests
   python backend/tests/smoke_tests.py
   ```

### 5.3 Post-Deployment Verification
- [ ] Backend service running
- [ ] Frontend accessible
- [ ] WebSocket connections working
- [ ] Database connections stable
- [ ] Exchange APIs responding
- [ ] Monitoring dashboards updated
- [ ] No errors in logs
- [ ] Sample strategy execution successful

### 5.4 Rollback Procedure
If deployment fails:
```bash
# Stop services
systemctl stop hustle-backend

# Restore code
git reset --hard <previous-commit-hash>

# Restore database
python backend/scripts/restore_database.py --backup <backup-file>

# Restore configuration
cp backend/config/production.yaml.backup backend/config/production.yaml

# Start services
systemctl start hustle-backend

# Verify rollback
python backend/tests/smoke_tests.py
```
